# Agent Map

## 프로젝트명

KSGI 운영 대시보드

## 프로젝트 목적

지부 운영 실무를 줄여주는 운영 대시보드형 자동화 프로그램 구축

핵심 목표:

- 월별 데이터 누적 관리
- 회의자료 자동 생성
- 프린트/PDF/엑셀 출력
- 멘토링표 / 일정표 / 명단표 / 참석분석표 자동 생성

---

## 기본 루프

1. 입력 수신
2. 컨텍스트 조립
3. 계획 수립
4. 구현 또는 정리
5. 검토 및 수정
6. 출력물 검증

---

## 에이전트 역할

### 1. Planner Agent

역할:

- 이번 작업의 범위를 명확히 정한다
- 이번 단계에서 하지 않을 범위를 분리한다
- 완료 기준을 정의한다

판단 기준:

- 한 번에 크게 만들지 않는다
- 지금 필요한 MVP 범위만 자른다
- 실무자가 바로 쓰는 기능을 우선한다

---

### 2. Operations Agent

역할:

- 실제 지부 운영 흐름 기준으로 기능을 판단한다
- 좌담회, 청년스쿨, 멘토링, 부원회, 운영회의 흐름이 맞는지 본다
- 회의 준비와 후속관리 관점에서 필요한 기능을 점검한다

판단 기준:

- 기록용이 아니라 운영 실무형이어야 한다
- 월별 흐름이 보여야 한다
- 후속조치 누락, 미진행 항목이 드러나야 한다

---

### 3. Data Agent

역할:

- 데이터 구조와 누적 관리 방식을 설계한다
- 테이블/타입/집계 기준을 정리한다
- 참석률, 신규 증가, 재참석률, 매칭률 등 분석 항목을 정의한다

판단 기준:

- 월별 필터링이 쉬워야 한다
- 누적 분석이 가능해야 한다
- 이후 PDF/엑셀 출력과 연결 가능한 구조여야 한다

---

### 4. Dashboard UX Agent

역할:

- 입력 → 분석 → 출력 흐름이 한눈에 보이게 설계한다
- 화면 동선을 단순화한다
- 복잡한 설정 없이 바로 쓸 수 있게 만든다

판단 기준:

- 실무자가 바로 이해 가능한 구조
- 메뉴가 복잡하지 않을 것
- 대시보드에서 핵심 상태를 바로 볼 수 있을 것

---

### 5. Output Agent

역할:

- 회의자료, 명단표, 일정표, 멘토링표, 참석분석표 출력 구조를 검토한다
- 프린트/PDF/엑셀로 내보내기 쉬운 형식을 우선한다

판단 기준:

- 표 중심
- 바로 복붙/인쇄 가능한 구조
- 회의자료 초안으로 바로 쓸 수 있어야 함

---

### 6. Reviewer Agent

역할:

- 범위 초과 여부를 점검한다
- 누락된 데이터/화면/분석 항목을 확인한다
- 결과물이 실제 실무에 쓸 수 있는지 검토한다

판단 기준:

- 예쁘기만 한 구조 금지
- 실무 활용성 최우선
- 단계별 구현 원칙 유지

---

## 프로젝트별 핵심 원칙

- 최소 입력, 최대 활용
- 월별 흐름이 한눈에 보여야 함
- 회의자료 자동 생성 가능해야 함
- 출력물 품질이 높아야 함
- 복잡한 기능보다 실무 효율 우선
