# KSGI 운영 대시보드 - 데이터 구조 설계 (Data Structure)

이 프로젝트는 브라우저 기반의 로컬 데이터베이스인 **Dexie (IndexedDB)**를 사용하여 데이터 보안과 오프라인 접근성을 보장합니다.

## 1. 데이터 모델 (Schema)

### Member (지부 부원)

- 지부의 인적 자원을 관리하는 중심 테이블입니다.
- **주요 필드**: `name`, `division` (여성부/미래부 등), `position`, `isYouth`, `mentorId` (멘토링 매칭).

### Meeting (활동 기록)

- 좌담회, 부원회, 청년스쿨, 운영회의 등 모든 공식 활동을 기록합니다.
- **주요 필드**:
  - `type`: '좌담회' | '부원회' | '청년스쿨' | '운영회의'
  - `date`: 문자열 (ISO 형식 또는 YYYY-MM-DD)
  - `attendees`: 참여 부원 ID 배열 (`number[]`)
  - `absentees`: 미참석자 관리용 ID 배열
  - `newAttendees`: 아직 DB에 등록되지 않은 신규 인원 이름
  - `minutes`: 회의록 및 주요 내용
  - `tasks`: 회의 후속 조치 사항 (`task`, `PIC`, `status`)
- **실무 활용**: 이 테이블의 데이터를 월별로 취합하여 '참석 분석표'와 '인쇄용 운영 보고서'를 자동 생성합니다.

### MentoringLog (멘토링 기록)

- 1:1 활동 및 상담 기록을 관리합니다.
- **주요 필드**: `mentorId`, `menteeId`, `date`, `note`.

## 2. 데이터 흐름 (Data Flow)

1. **입력(Capture)**: OCR 이미지 또는 폼 입력을 통해 `Meeting`과 `Member` 데이터가 생성됩니다.
2. **저장(Persist)**: Dexie를 통해 브라우저에 즉시 저장됩니다.
3. **가공(Process)**: 대시보드 및 보고서 컴포넌트에서 월별/부서별 통계로 가공됩니다.
4. **출력(Export)**: `@media print` 스타일이 적용된 UI를 통해 인쇄되거나 PDF/엑셀로 변환됩니다.

## 3. 확장 고려 사항

- **이미지 데이터**: OCR 분석 후 원본 이미지를 저장할 경우 브라우저 용량 제한을 고려하여 외부 저장소 연동이나 최적화가 필요할 수 있습니다.
- **백업/복원**: 사용자가 브라우저를 초기화하더라도 데이터를 보존할 수 있도록 JSON 형태의 백업/복원 기능을 구현할 예정입니다.
