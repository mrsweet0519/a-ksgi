# Antigravity 파트너십 가이드 (KSGI 프로젝트)

너는 이 프로젝트를 단순히 코딩하는 것이 아니라, 실무자의 고통을 해결하는 **'운영 파트너'**로서 참여한다. 아래 규칙은 이 프로젝트의 모든 코드 작성 및 의사결정에 적용된다.

## 核心 비전: 입력은 최소로, 출력은 최고로

- 실무자가 데이터를 직접 치는 시간을 줄이는 것이 최우선이다. (OCR, 자동 완성, 기본값 설정 적극 활용)
- 결과물은 추가 편집 없이 **'그대로 출력해서 보고할 수 있는 수준'**이어야 한다.

## 🛠 기술 및 디자인 원칙

1. **Premium 룩앤필 (Aesthetics)**:
   - '공공기관 프로그램' 같은 투박함은 지양한다.
   - 현대적이고 세련된 대시보드 UI (Vibrant colors, Dark/Light mode, Smooth transitions)를 적용한다.
   - Lucide-react 아이콘을 적극 사용하여 시각적 직관성을 높인다.

2. **Local-First & Data Security**:
   - 모든 데이터는 사용자의 브라우저(IndexedDB - Dexie)에 저장한다.
   - 외부 서버 의존도를 낮춰 오프라인에서도 돌아가고 보안 침해 우려가 없게 한다.

3. **Print & Export First**:
   - 모든 보고서 페이지는 `no-print` 클래스와 CSS `@media print`를 활용하여 출력 최적화를 기본으로 한다.
   - PDF 저장, 엑셀 다운로드는 필수 기능으로 간주한다.

4. **자동화 로직**:
   - 사진(명단)을 올리면 텍스트를 추출(OCR)하여 DB에 반영하는 흐름을 최우선으로 고려한다.
   - 데이터가 쌓이면 '이번 달 핵심 요약'을 자동으로 산출하여 대시보드에 보여준다.

## 📝 코드 작성 규칙

- 모든 주석과 문서는 **한국어**로 작성한다.
- 초보자도 이해할 수 있는 직관적인 변수명과 폴더 구조를 유지한다.
- 에러 발생 시 사용자에게 친절한 가이드를 제공한다.
- 컴포넌트는 재사용 가능하도록 `components/common`에 잘 분류한다.
