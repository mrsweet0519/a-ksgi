# KSGI Project Instructions

## 프로젝트 목적

이 프로젝트는 KSGI 계수지부 여성부장 업무를 돕기 위한 지부 운영 대시보드 프로젝트이다.

주요 목적은 다음과 같다.

- 매월 좌담회 참석 현황을 한눈에 파악한다.
- 좌담회 참석자, 미참석자, 신규 참석자, 후속 연락 대상을 정리한다.
- 창가청년스쿨 참석자와 현장 반응을 관리한다.
- 청년 멘토링, 장남일체, 부녀일체 기반의 1:1 근행회 연결 상태를 관리한다.
- 부서별 부원회, 미래부 부원회, 운영회의 자료를 정리한다.
- 월간 회의자료, 보고서, 일정표, 카카오톡 공유용 이미지를 만들 수 있도록 지원한다.

## 사용자 기준

사용자는 개발 초보자이다.

작업할 때는 다음 기준을 지킨다.

- 어려운 개발 용어는 쉽게 풀어서 설명한다.
- 파일을 수정할 때는 어느 파일을 왜 수정했는지 알려준다.
- 명령어만 나열하지 말고 실행 결과와 다음 조치를 함께 설명한다.
- 한 번에 너무 많은 기능을 만들지 말고 단계별로 진행한다.
- 화면 기준 설명이 필요한 경우 클릭 위치와 확인 방법을 구체적으로 설명한다.

## 화면 구성 원칙

- 첫 화면에는 월간 핵심 요약을 보여준다.
- 좌담회 참석률, 신규 참석자 수, 후속 연락 필요 인원, 멘토링 진행률을 카드 형태로 보여준다.
- 최근 3개월 흐름을 볼 수 있게 한다.
- 표는 인쇄와 회의자료 활용이 가능하도록 정리한다.
- 모바일보다 PC와 출력물 기준 가독성을 우선한다.
- 카카오톡 공유용 이미지는 버튼이나 편집 도구가 보이지 않는 깨끗한 저장본을 목표로 한다.

## 기능 개발 우선순위

1. 현재 화면 오류 확인
2. Recharts 차트 크기 경고 해결
3. 새 일정 등록 버튼 동작 확인
4. 좌담회 관리 화면 개선
5. 월간 보고서 생성 기능 점검
6. 엑셀 다운로드 기능 점검
7. 회의자료 초안 생성 기능 점검
8. 카카오톡 공유용 이미지 저장 품질 개선
9. 데이터 입력/수정 흐름 정리
10. 전체 대시보드 사용성 개선

## Playwright 테스트 기준

Playwright MCP가 연결되어 있으므로 주요 수정 후에는 실제 브라우저로 확인한다.

확인 기준은 다음과 같다.

- localhost 접속 성공 여부
- 화면 렌더링 정상 여부
- 콘솔 오류 여부
- 콘솔 경고 여부
- 버튼 클릭 가능 여부
- 모달 또는 화면 전환 여부
- 주요 메뉴 이동 여부
- 저장/다운로드 기능 작동 여부
- 화면 깨짐 여부

단, 다운로드, 삭제, 데이터 변경, 보고서 생성처럼 파일이나 데이터에 영향을 줄 수 있는 기능은 실행 전 사용자 확인을 받는다.

## 안전 기준

다음 작업은 반드시 사용자 확인 후 진행한다.

- 파일 삭제
- 기존 파일 전체 덮어쓰기
- 데이터 초기화
- Git push
- DB 쓰기 작업
- 토큰 또는 API Key 변경
- 환경변수 영구 변경
- 대량 파일 이동
- 의도하지 않은 다운로드 실행

## 코드 수정 기준

- 기존 기능을 먼저 파악한 뒤 수정한다.
- 새 기능을 추가하기 전에 관련 파일 구조를 확인한다.
- 기존 코드 스타일을 최대한 유지한다.
- 한 번에 여러 파일을 크게 바꾸지 않는다.
- 수정 후 반드시 npm run dev 또는 관련 테스트로 확인한다.
- 오류가 나면 원인, 수정 내용, 결과를 정리한다.

## 보고 방식

작업 결과는 다음 형식으로 보고한다.

1. 작업한 파일
2. 수정한 이유
3. 실제 수정 내용
4. 실행한 명령어
5. 확인 결과
6. 남은 문제
7. 다음 추천 작업

## 현재 우선 점검 항목

Playwright MCP 테스트 결과, 다음 항목을 우선 확인한다.

- Recharts 경고 원인 확인
- 새 일정 등록 버튼 클릭 후 모달 또는 화면 변화가 없는 문제 확인
- 5173과 5174 포트가 동시에 응답하는 상태 정리
- 월간 보고서 생성, 엑셀 다운로드, 회의자료 초안 생성 버튼의 안전한 테스트 방식 정리
